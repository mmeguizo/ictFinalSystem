1. System Overview

The ICT Support Ticketing System is a web-based service request platform designed for the ICT Department of CHMSU. It enables employees to submit service requests, which are routed through a structured approval and assignment workflow, tracked with SLA enforcement, and enhanced by AI-powered analysis.

Feature Description
Ticket Management Submit, track, and resolve IT service requests (MIS and ITS)
Approval Workflow Multi-level review: Secretary → Director → Department Head
SLA Enforcement 5-step, 25-minute processing time target with automated breach alerts
AI Analysis Google Gemini-powered ticket analysis with smart suggestions
Knowledge Base Self-service FAQ library to resolve common issues
Real-Time Updates WebSocket-driven live dashboard and notifications
Analytics Charts, trends, SLA compliance, staff performance, PDF/Excel export
Satisfaction Survey Star rating feedback after ticket resolution

3. Dashboard

The Dashboard is the first page after login. It provides a real-time overview of ticket activity.

Statistics Cards

Live-updating stat cards: Total Tickets, For Review, Reviewed, Assigned, In Progress, On Hold, Resolved, Closed, Cancelled, Ongoing.

  • Cards update in real-time via WebSocket — no manual refresh needed
  • A green Live indicator appears when the WebSocket connection is active
  • Cards pulse briefly when their values change

SLA Reminder

On login, regular users (USER role) see a popup explaining the 5-step SLA processing time (25 minutes total). The "Learn More" button on the banner is available to all roles.

Recent Tickets

Table showing the last 10 tickets created. Click any row to navigate to the ticket detail page.

4. Submitting a Ticket

Who can submit: USER, SECRETARY, ADMIN

Choose Ticket Type

Type Purpose Routed To
MIS Website and software development requests MIS_HEAD → DEVELOPER
ITS Equipment borrowing and maintenance requests ITS_HEAD → TECHNICAL

Common Fields

  • Title (required) — Brief description of the issue (min 5 characters)
  • Description (required) — Detailed explanation (min 10 characters)
  • Priority (optional) — LOW, MEDIUM, HIGH, or CRITICAL (default: MEDIUM)

MIS-Specific Fields

Category (WEBSITE or SOFTWARE), checkboxes for New Request / Update / Install, Website URL, Estimated Duration, Additional Notes.

ITS-Specific Fields

Device Type, Device Model, Serial Number, Borrow Request + Details, Maintenance checkboxes + Details.

Auto-Generated Fields

  • Ticket Number: TKT-YYYYMMDD-NNN (e.g., TKT-20260313-001)
  • Control Number: YYYY-MM-NNN (e.g., 2026-03-001)
  • Due Date: CRITICAL=4h, HIGH=24h, MEDIUM=72h, LOW=168h (7 days)

7. SLA Processing Time Tracker

Located on the Ticket Detail Page. Target: 25 minutes total (5 min per step).

1
Secretary Review Secretary reviews and endorses the ticket (5 min)
2
Director Endorsement Director reviews and approves the request (5 min)
3
Assignment Auto-assigned to department head, then to specific staff (5 min)
4
Schedule Visit Head sets visit date and target completion (5 min)
5
Acknowledgment Admin acknowledges the schedule, staff starts work (5 min)

SLA Breach Alerts

  • A cron job runs every 5 minutes to detect overdue tickets
  • Level 1: Notifies assigned staff + department head
  • Level 2: Notifies ADMIN + DIRECTOR

8. Approval Workflow

Secretary Workflow

  1. View tickets with status FOR_REVIEW on the Approvals page
  2. Click "Review" to endorse → status becomes REVIEWED
  3. Or click "Reject" with reason → status becomes CANCELLED

Director Workflow

  1. View tickets with status REVIEWED
  2. Click "Approve" → status becomes DIRECTOR_APPROVED
  3. Or click "Disapprove" with reason → status becomes CANCELLED

9. Assignment & Scheduling

Assigning Staff (MIS_HEAD / ITS_HEAD)

  1. After director approval, the ticket auto-arrives in the department head's queue
  2. Open the ticket and click "Assign to Staff"
  3. Select a staff member from the dropdown (sorted by workload)

Scheduling a Visit

  1. Click "Schedule Visit"
  2. Set the Date to Visit and Target Completion Date
  3. Add monitor notes (optional) → status becomes PENDING_ACKNOWLEDGMENT

Admin Acknowledgment

Admin reviews scheduled visits and clicks "Acknowledge" → status becomes SCHEDULED.

10. Working on Tickets (DEVELOPER / TECHNICAL)

  1. Start Work — When ticket is SCHEDULED, click "Start Work" → IN_PROGRESS
  2. Add Notes — Document progress, troubleshooting steps, findings
  3. Upload Attachments — Screenshots, logs, documents
  4. Put On Hold — If waiting for parts/info → ON_HOLD
  5. Resume — When ready → back to IN_PROGRESS
  6. Resolve — Work complete, add final comment → RESOLVED

11. Admin Functions

  • User Management — Create, edit, delete user accounts and assign roles

  • Close Tickets — Finalize resolved tickets

14. Notifications

Notification Bell

Bell icon in the navigation bar shows unread count badge. Click to see last 5 notifications. Click a notification to navigate to the related ticket.

Notification Events

Event Who Gets Notified
New ticket submitted Secretary, Admin
Ticket reviewed Ticket creator, Director
Ticket rejected/disapproved Ticket creator
Ticket approved Department Head, Ticket creator
Staff assigned Assigned staff, Ticket creator
Status changed All ticket participants
Note/attachment added All ticket participants
SLA breach Staff, Head, Admin

Notifications are delivered in real-time via WebSocket with 30-second polling fallback.

15. AI-Powered Smart Suggestions

Powered by Google Gemini 2.0 Flash.

How to Use

  1. On the Submit Ticket page, fill in Title and Description
  2. Click "Analyze with AI"
  3. Wait for AI processing

What You Get

Output Description
Clean Ticket AI-rewritten, professional version of your description
Summary Brief one-sentence summary
Category Suggested category (Network, Software, Hardware, etc.)
Priority Suggested priority with confidence score
Root Cause Analysis of the underlying problem
Solutions Suggested next steps or solutions
Keywords Extracted keywords for searching similar issues

Click "Apply to Description" to use the AI-rewritten description. The system also shows similar resolved tickets and related Knowledge Base articles.

16. Satisfaction Survey

After your ticket is RESOLVED or CLOSED, a survey prompt appears on the ticket detail page.

  1. Select a star rating (1 to 5 stars)
  2. Optionally add a comment
  3. Click Submit

17. Roles & Ticket Status Reference

System Roles

Role Responsibility
USER Submit tickets, track progress, rate satisfaction
SECRETARY Review and endorse incoming tickets
DIRECTOR Approve or disapprove reviewed tickets
MIS_HEAD Assign MIS tickets to developers, schedule visits
ITS_HEAD Assign ITS tickets to technicians, schedule visits
DEVELOPER Work on MIS tickets (software/web development)
TECHNICAL Work on ITS tickets (hardware/network support)

Ticket Statuses

Status Meaning Who Acts Next
